Так мужики, пишем сайт наконец-то, а то у нас неделя до защиты. Пишем все таки на GO, потому что потому.
Давайте соблюдаем базовую структуру (помогал Claude):
```
literary_club\
    cmd\
      web\main.go - Основная логика, точка входа
    internal\     - Весь приватный код
    web\          - Весь фронт
    migrations\   - SQL-миграции (клод сказал так надо, видать будет БДшка какая-то)
    config\       - Настройки, .env (хуй знает что это значит)
    go.mod
    go.sum
    README.md

Да придется соблюдать струкуру, а для этого нужно вспомнить, что было на парах по GO.
```
---------------------------------------------------------------------------------------------------------------

Работаем по стандартной схеме:
Делаем форк -> Клонируем реп -> Добавляете этот реп как upstream -> Делаем ветку и работаем в общей структуре (отдельные папки не создаем, меняем существующие файлы, либо создаем новые, либо отдельные папка, НО ВСЕ В ОБЩЕЙ СТРУКТУРЕ) -> дальше пушим к себе в форк -> PR и т.д.

Если что-то новое уже кто-то сделал, то делаем pull -> merge -> работаем -> push

Т.к. репозиторий мой, то я работаю сразу в нем, а также принимаю все PR

---------------------------------------------------------------------------------------------------------------

Папки пока не отображаются, но как только будете писать, просто у себя создайте в соответствии структуре, затем в них работайте, потом уже они будут у всех.
Как писать разбирайтесь самостоятельно, делайте сборку и т.д. как на GO учили, в go.mod все будет добавляться само.

В main НЕ РАБОТЕМ, ТОЛЬКО В ОТДЕЛЬНЫХ ВЕТКАХ. 
Коммиты называйте осмысленно [тип коммита: ][что именно сделали]. Пример: "feat: add user registration" или "bugfix: fix DB requesting"

---------------------------------------------------------------------------------------------------------------
